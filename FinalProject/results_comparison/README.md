# Results Comparison

This folder compares the output filter matrices from all implementations against the Python reference output.

This comparison is currently documented for the following case:

```text
H    = 64
W    = 64
Cin  = 3
Cout = 8
K    = 3
```

That means:

```text
Input data  = 3 input channels, each 64 × 64
Kernel data = 8 filters × 3 input channels × 3 × 3 weights
Output data = 8 output matrices, each 64 × 64
```

## Project folder names assumed

This comparison script assumes the main project directory contains:

```text
sequential_implementation/
openmp/
cuda_naive/
cuda_shared/
python_ref/
inputs/
results_comparison/
```

The Python reference is used as the correctness reference.

## Required files

Before running comparison, each implementation folder must already contain its output matrices.

Expected files:

```text

sequential_implementation/sequential_filter_1.csv
...
sequential_implementation/sequential_filter_7.csv

openmp/openmp_filter_0.csv

...

cuda_naive/cuda_naive_filter_0.csv
cuda_naive/cuda_naive_filter_1.csv

cuda_shared/cuda_shared_filter_0.csv

python_ref/python_reference_filter_0.csv
python_ref/python_reference_filter_1.csv
...
python_ref/python_reference_filter_7.csv
```

The folders should also contain the same input data files:

```text
input.csv

```

These files should originally be generated in the `inputs/` folder, then copied to each implementation folder:

```text
sequential_implementation/

cuda_naive/
cuda_shared/
python_ref/
```

The names must be exactly:

```text
input.csv
kernel.csv
```

## What the comparison checks

The comparison performs two checks.

### 1. Input/kernel consistency

The script checks whether `input.csv` and `kernel.csv` are byte-identical across all folders.

It writes:

```text
input_kernel_consistency.csv
```

### 2. Output matrix correctness

For each filter, it compares:

```text
sequential_filter_i.csv     vs python_reference_filter_i.csv
openmp_filter_i.csv         vs python_reference_filter_i.csv
cuda_naive_filter_i.csv     vs python_reference_filter_i.csv
cuda_shared_filter_i.csv    vs python_reference_filter_i.csv
```

It computes:

```text
max_abs_diff
mean_abs_diff
rmse
number of values above tolerance
```

The default tolerance is:

```text
1e-4
```

A result passes if:

```text
max_abs_diff <= 1e-4
```

## Output files generated by this folder

After running comparison, this folder generates:

```text
comparison_report.txt
comparison_summary.csv
input_kernel_consistency.csv
comparison.out
comparison.err
```

## Why not use Linux diff?

Do not use plain `diff` for floating-point matrix files.

For example:

```text
0.12345678
0.123457
```

may be numerically close but textually different.

This folder uses numerical comparison with tolerance, which is the correct method for comparing floating-point results.
